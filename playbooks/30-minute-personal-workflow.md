# 30 分钟个人工作流改造

## 目标

用 30 分钟把一个真实任务从“临时问 AI”改造成“有目标、有上下文、有验证、有复盘”的小型工作流。

## 适合谁

- 刚开始系统使用 AI 的个人。
- 已经会用 AI，但产出质量不稳定的人。
- 想把一个重复任务模板化的人。

## 预计耗时

30 分钟。

## 最终产出

- 一份任务说明。
- 一份上下文清单。
- 一份验证清单。
- 一条可复用经验。

## 第 1 步：选择一个真实任务，5 分钟

选择你本周一定要完成的任务，例如：

- 写一篇文章。
- 分析一批用户反馈。
- 设计一个功能方案。
- 修改一个页面。
- 准备一次会议。

写下：

- 任务名称：
- 目标用户：
- 成功标准：
- 截止时间：

## 第 2 步：整理上下文，8 分钟

不要直接让 AI 开始做，先给它必要材料。

上下文至少包含：

- 背景：为什么要做？
- 约束：不能做什么？
- 样例：什么结果算好？
- 风格：表达或实现要符合什么标准？
- 风险：哪里最容易出错？

## 第 3 步：设计人机分工，5 分钟

写下 AI 负责什么、人负责什么。

| 环节 | AI 负责 | 我负责 |
| --- | --- | --- |
| 理解问题 |  |  |
| 生成初稿 |  |  |
| 找问题 |  |  |
| 最终判断 |  |  |

## 第 4 步：执行并保留证据，7 分钟

让 AI 输出结果时，同时要求它说明：

- 它用了哪些假设。
- 哪些地方不确定。
- 哪些地方需要你确认。
- 哪些结果可以被检查。

## 第 5 步：验证和复盘，5 分钟

检查：

- 是否解决了最初目标？
- 是否违反约束？
- 是否需要人工审核？
- 是否有明显遗漏？
- 是否能沉淀成下次模板？

最后写一句复盘：

> 下次遇到类似任务，我会先提供 ______，并用 ______ 检查结果。
