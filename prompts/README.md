# Prompt 使用原则

本仓库不把 Prompt 当作核心能力。Prompt 很重要，但它只是工作流的一部分。

更可靠的顺序是：

```text
目标 -> 上下文 -> 任务拆解 -> Prompt -> 输出 -> 验证 -> 复盘
```

## 好 Prompt 的结构

一个可复用 Prompt 通常包含：

- 角色：你希望 AI 从什么视角工作。
- 目标：这次要解决什么问题。
- 上下文：它必须知道哪些背景。
- 输入：你提供了哪些材料。
- 输出格式：你希望它如何返回。
- 约束：哪些事情不要做。
- 验证：它需要如何自检。

## 模板

```text
你现在扮演 [角色]。
我的目标是 [目标]。
背景信息是 [上下文]。
请基于以下输入工作：[输入]。
输出格式请使用 [格式]。
不要做 [限制]。
最后请列出你不确定的地方，以及我应该如何验证结果。
```

## 反向检查 Prompt

```text
请不要继续扩写。请只做审查：
1. 这个结果哪里可能是错的？
2. 哪些地方缺少证据？
3. 哪些表达太空泛？
4. 哪些结论需要人工确认？
5. 如果要交付给真实用户，最低还要补什么？
```

## 使用原则

- 同一个任务的好 Prompt 应该能被下次复用。
- Prompt 不能替代业务判断。
- 重要任务必须要求 AI 暴露不确定性。
- 结果必须经过人、测试、样例或指标验证。
